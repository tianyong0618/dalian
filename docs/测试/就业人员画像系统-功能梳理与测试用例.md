# 就业人员画像系统 - 核心功能梳理与测试用例

## 一、系统概述

就业人员画像系统是一个基于多源数据融合的人才分析与管理系统，通过整合社保、就业、人才、权益等数据，构建全面的人员画像，为政府决策、企业招聘、个人发展提供数据支撑。

---

## 二、核心功能模块

### 2.1 数据概览模块

| 功能项 | 说明 | 数据来源 |
|--------|------|----------|
| **人员总数统计** | 显示系统内就业人员总数 | 多源数据聚合 |
| **标签数量统计** | 显示已定义的标签数量 | 标签管理系统 |
| **画像准确度** | 显示画像数据的准确率百分比 | AI算法评估 |
| **群体分组数** | 显示已划分的人员群体数量 | 聚类分析结果 |

### 2.2 数据源管理模块

| 数据源 | 数据内容 | 管理功能 |
|--------|----------|----------|
| **社保数据** | 参保记录、缴费基数、缴费年限 | 数据接入、质量监控 |
| **就业数据** | 就业登记、失业登记、求职记录 | 数据同步、异常检测 |
| **人才数据** | 学历信息、技能证书、培训记录 | 数据清洗、标签提取 |
| **权益数据** | 劳动合同、工资记录、维权记录 | 数据关联、风险预警 |

### 2.3 人员检索与筛选模块

#### 2.3.1 搜索功能
- **关键词搜索**：支持姓名、身份证号、标签关键词搜索
- **实时筛选**：输入时实时过滤结果

#### 2.3.2 筛选条件
| 筛选维度 | 选项值 |
|----------|--------|
| **群体分组** | 全部、核心人才、零工人员、就业困难人员、大龄求职者、高校毕业生 |
| **学历层次** | 全部、博士、硕士、本科、大专、高中及以下 |
| **就业稳定性** | 全部、稳定(>70分)、一般(40-70分)、不稳定(<40分) |
| **画像类型** | 全部、人才画像、求职者画像 |

### 2.4 画像生成模块

#### 2.4.1 一键生成画像流程
```
步骤1: 选择人员 → 步骤2: 数据采集 → 步骤3: 标签生成 → 步骤4: 完成
```

#### 2.4.2 画像维度评分
| 维度 | 说明 | 评分范围 |
|------|------|----------|
| **教育竞争力** | 学历、专业、学校层次综合评分 | 0-100% |
| **就业稳定性** | 工作年限、跳槽频率、社保连续性 | 0-100% |
| **职业技能** | 技能证书、培训经历、专业能力 | 0-100% |
| **薪资趋势** | 薪资增长幅度、行业对比 | 0-100% |

### 2.5 标签管理模块

#### 2.5.1 标签分类
| 分类 | 标签示例 |
|------|----------|
| **基础信息标签** | 年龄：25-35岁、大专学历、本地户籍 |
| **教育特征标签** | 本科及以上、计算机专业、教育竞争力：高 |
| **就业特征标签** | 社保连续缴纳、就业稳定性：高、工作年限：3-5年 |
| **技能标签** | Java开发、Python、SQL |
| **帮扶标签** | 就业困难人员、帮扶等级：C级、需技能培训 |
| **意向标签** | 求职中、在职观望、创业意向 |

#### 2.5.2 标签操作
- **手动添加**：输入标签名称，自定义添加
- **AI智能生成**：系统自动分析数据生成标签
- **标签删除**：支持单个标签删除

### 2.6 人员详情模块

#### 2.6.1 基础信息展示
- 个人照片/头像
- 姓名、性别、年龄、学历
- 联系方式
- 身份标签（核心人才、技术骨干等）
- 求职状态（在职求职、在职稳定等）

#### 2.6.2 多维度画像Tab

| Tab名称 | 内容说明 |
|---------|----------|
| **基础信息** | 个人信息、联系方式、户籍信息 |
| **教育背景** | 学历信息、学校、专业、学位、继续教育记录 |
| **职业经历** | 工作经历时间轴、企业信息、职位、工作年限统计 |
| **核心能力** | 专业技能、通用能力、技能证书 |
| **就业服务** | 帮扶等级、服务需求、服务历史记录 |
| **薪资分析** | 薪资水平、薪资趋势、薪资结构、福利待遇 |
| **人岗匹配** | 推荐岗位、匹配度评分、技能匹配分析 |
| **风险评估** | 失业风险等级、风险因素、发展潜力评估 |

| **帮扶等级** | 根据就业困难程度划分的等级(A/B/C/D) |
