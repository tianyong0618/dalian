# 调解仲裁服务平台PRD功能增强设计

**日期**：2026-05-08
**基于**：`docs/prd/劳动人事争议"无接触"调解仲裁服务平台功能需求说明.docx`
**改造文件**：`原型设计/14-mediation-arbitration.html`
**方案**：增量增强（方案A）

---

## 一、概述

### 改造目标
根据PRD需求文档，对现有原型页面进行功能增强，覆盖7大核心模块：
1. 电子劳动合同管理 - 备案编号+数据联动
2. 根治欠薪一体化平台 - 已基本覆盖，小幅优化
3. 在线调解仲裁 - AI校验+案件分流
4. 在线劳动人事争议调解 - 智能分配+AI辅助
5. 数字化仲裁庭 - 流程自动引导
6. 仲裁工作上下联动 - 效能报表增强
7. 系统管理 - 打印+导出功能

### 方案对比
| 方案 | 策略 | 优点 | 缺点 | 推荐 |
|------|------|------|------|------|
| A：增量增强 | 在现有HTML基础上增加功能 | 快速交付，风险低，保留现有逻辑 | HTML变长 | ✅ |
| B：模块化重构 | 拆分为独立组件 | 结构清晰，易维护 | 工作量大，需重构 | |
| C：完全重写 | 采用现代框架重写 | 代码最优，性能最佳 | 周期长，偏离原型目标 | |

## 二、电子劳动合同模块增强

### 修改内容
1. **备案编号生成与展示**
   - 合同签署后自动生成唯一备案编号（格式：HT + 日期 + 4位随机数）
   - 示例：HT202605080001
   - UI位置：合同签署模态框，在"生成合同"按钮下方显示备案信息

2. **数据联动状态展示**
   - 增加"数据联动状态"区域，显示：
     - ✓ 已同步至根治欠薪平台
     - ✓ 已同步至调解仲裁平台
     - ✓ 作为用工关系认定依据

### 修改位置
- 文件：`14-mediation-arbitration.html`
- 第518-583行：电子劳动合同模态框（ec-contract-modal）
- 在"生成合同"按钮逻辑中，增加备案编号生成
- 在模态框中增加备案信息展示区域

## 三、在线仲裁模块增强

### 修改内容
1. **AI材料校验提示**
   - 位置：在线立案第2步（上传证据）
   - 校验项：材料完整性、格式合规性、证据链完整性
   - UI形式：黄色提示框，列出校验结果

2. **案件分流提示**
   - 位置：确认提交页面（第3步）
   - 分流逻辑：根据案件类型、金额、复杂程度自动判断
   - 展示内容：
     - 建议分流至：仲裁程序/调解程序
     - 分流依据：争议金额、案件类型、证据完整性

### 修改位置
- 第336-462行：在线立案模态框（case-register-modal）
- 第2步（reg-form-2）：增加AI校验结果展示
- 第3步（reg-form-3）：增加案件分流建议展示

## 四、在线调解模块增强

### 修改内容
1. **调解员智能推荐**
   - 位置：调解员分配模态框
   - 推荐依据：专长匹配度、工作负荷、历史成功率、评分
   - UI形式：绿色提示框，展示AI推荐结果和备选方案

2. **调解过程AI辅助**
   - 位置：在线调解室模态框
   - 功能：争议焦点分析、调解协议模板生成
   - UI形式：在调解沟通区增加AI辅助面板

### 修改位置
- 第1058-1120行：调解员分配模态框（mediation-assign-modal）
- 第1122-1221行：调解室模态框（mediation-room-modal）
