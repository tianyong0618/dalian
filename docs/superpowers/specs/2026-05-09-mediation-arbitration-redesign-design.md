# 调解仲裁服务平台重构设计规格

## 1. 项目概述

**项目名称**：劳动人事争议"无接触"调解仲裁服务平台  
**设计日期**：2026-05-09  
**核心目标**：按照PRD要求重新设计调解仲裁服务原型，支持五类用户角色独立界面，完整呈现云仲裁、AI智能应用功能。

## 2. 设计方案

### 2.1 整体架构

| 文件名 | 说明 |
|--------|------|
| 14-mediation-arbitration.html | 平台首页（五角色统一入口/概览仪表盘） |
| role-laborer.html | 劳动者端 |
| role-employer.html | 用人单位端 |
| role-arbitration.html | 仲裁机构端 |
| role-supervisor.html | 人社监管端 |
| role-legal-aid.html | 法律援助端 |

**角色切换**：各页面顶部导航栏包含角色切换下拉菜单，支持快速切换不同角色视角。

### 2.2 统一UI规范

- **样式文件**：复用 `原型设计/style.css`
- **颜色变量**：--primary (#1677ff)、--success (#52c41a)、--warning (#faad14)、--danger (#ff4d4f)
- **圆角**：--radius-sm (6px)、--radius-md (12px)、--radius-lg (20px)
- **阴影**：遵循现有shadow系统

### 2.3 各角色功能模块

#### 劳动者端 (role-laborer.html)
- 仪表盘：我的案件统计、待处理事项
- 电子劳动合同：我的合同、合同签署
- 案件申请：在线提交仲裁申请、材料上传
- 我的案件：调解/仲裁进度查看、云庭审参与
- 文书查看：已送达文书下载、签收确认
- 法律援助：援助申请入口

#### 用人单位端 (role-employer.html)
- 仪表盘：合同统计、案件统计
- 合同管理：发起签署、合同列表、合同备案查询
- 应诉案件：收到的仲裁通知、答辩材料提交
- 考勤上报：工程项目考勤数据上传
- 工资确认：工资发放记录提交、农民工确认查询
- 欠薪投诉处理：欠薪投诉响应

#### 仲裁机构端 (role-arbitration.html)
- 仪表盘：案件量统计、调解率、仲裁效率
- 案件管理：受理审核、材料校验、AI分流
- 调解管理：调解员分配、在线调解主持
- 云庭审：完整五分区布局（视频会议、证据展示、笔录区、操作栏、电子签字）
- 文书管理：裁决书/调解书生成、雲送达
- 上下联动：跨区域案件协同、数据同步状态

#### 人社监管端 (role-supervisor.html)
- 仪表盘：全市案件概览、预警统计
- 项目监管：在建工程项目全要素监管
- 欠薪预警：AI预警列表、预警级别、处理状态
- 效能统计：各级仲裁机构办结率、云庭审使用率
- 报表中心：监管报表导出

#### 法律援助端 (role-legal-aid.html)
- 仪表盘：援助案件统计、待审核
- 援助申请：申请审核、资格核验
- 案件管理：指派律师、服务记录
- 统计分析：援助覆盖率统计

### 2.4 云庭审五分区布局

```
┌──────────────────────────────────────────────────────────────┐
│  庭审信息栏：案件编号 | 案件类型 | 状态 | 开庭时间 | 倒计时    │
├─────────────────────┬──────────────────┬─────────────────────┤
│                     │                  │                     │
│   视频会议区         │   证据展示区      │   庭审笔录区        │
│   - 16:9主画面       │   - 材料实时共享  │   - 语音转文字      │
│   - 仲裁员/当事人    │   - 证据列表     │   - 实时更新        │
│   - 静音/摄像头控制   │   - 质证标记     │   - 历史记录        │
│                     │                  │                     │
├─────────────────────┴──────────────────┴─────────────────────┤
│  操作栏：[上传证据] [举证质证] [补充发言] [休庭] [电子签字]   │
└──────────────────────────────────────────────────────────────┘
```

### 2.5 AI功能可视化

| 功能 | 呈现方式 |
|------|----------|
| 材料校验 | 申请表单实时校验状态图标（✓/⚠️/✗）|
| 案件分流 | 提交后弹窗显示AI推荐及理由（调解优先/仲裁处理）|
| 欠薪预警 | 红色悬浮卡片+推送通知+红色预警级别标识 |
| 智能分配 | 调解员选择时显示AI推荐匹配度评分 |

### 2.6 技术要求

- **前端框架**：纯HTML/CSS/JS，复用现有style.css
- **数据模拟**：使用JS对象模拟各角色数据
- **交互实现**：完整的功能交互（模态框、标签页、表单提交）
- **响应式**：支持1280px以上宽度

## 3. 实现优先级

1. **Phase 1**：重构14-mediation-arbitration.html为五角色统一入口
2. **Phase 2**：实现劳动者端(role-laborer.html)
3. **Phase 3**：实现用人单位端(role-employer.html)
4. **Phase 4**：实现仲裁机构端(role-arbitration.html)含云庭审
5. **Phase 5**：实现人社监管端(role-supervisor.html)含预警
6. **Phase 6**：实现法律援助端(role-legal-aid.html)

## 4. 验收标准

- [ ] 五类用户角色可切换，各自功能模块完整
- [ ] 云庭审五分区布局完整呈现
- [ ] AI材料校验、案件分流、欠薪预警可视化展示
- [ ] 完整业务流程闭环：申请→审核→调解/仲裁→云庭审→文书送达→归档
- [ ] 上下联动数据贯通可视化
- [ ] 复用现有style.css样式规范
