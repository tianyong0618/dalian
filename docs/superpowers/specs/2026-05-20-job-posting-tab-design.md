# 发布岗位 Tab - 设计文档

## 概述

在人力资源服务旗舰店（`12-hr-flagship.html`）的"服务机构"角色下，新增"发布岗位"Tab，提供岗位发布、视频面试、政策咨询三个功能模块。

## 所属模块

- **项目**: 人社服务数字化平台原型
- **入口文件**: `原型设计/12-hr-flagship.html`
- **角色**: 服务机构（org）
- **Tab ID**: `org-job`
- **Tab 图标**: `fa-briefcase`
- **Tab 标签**: 发布岗位

## 页面布局（上下分区式）

### 1. 统计概览区（4个统计卡片）

| 卡片 | 说明 | 示例值 |
|------|------|--------|
| 在招岗位 | 当前在招岗位总数 | 18 |
| 本月发布 | 本月新发布岗位数 | 8 |
| 待面试 | 待安排的面试数 | 12 |
| 政策资料 | 可查阅政策资料数 | 36 |

复用现有的 `.stats-grid` 和 `.stat-card` 组件。

### 2. 岗位发布与管理

**搜索与操作栏**:
- 搜索输入框：按岗位名称/企业名称搜索
- "发布新岗位"按钮 → 弹窗表单

**筛选标签**: 全部 | 在招 | 已下线 | 待审核（复用 `.filter-tabs`）

**表格列**:
| 字段 | 说明 |
|------|------|
| 岗位名称 | 招聘岗位名称 |
| 招聘类型 | 高端人才/劳务派遣/灵活用工 |
| 发布企业 | 发布该岗位的企业名称 |
| 招聘人数 | 人数 |
| 发布时间 | 日期 |
| 状态 | 在招/已下线/待审核（badge） |
| 操作 | 查看/编辑/下线/安排面试 |

**发布新岗位弹窗**:
- 岗位名称（必填）
- 招聘类型（下拉：高端人才招聘/劳务派遣/灵活用工/人事外包）
- 发布企业（下拉：选择企业或自定义输入）
- 招聘人数
- 薪资范围
- 岗位描述（textarea）
- 学历要求
- 工作经验要求
- 截止日期

### 3. 视频面试

**操作栏**:
- "创建面试"按钮 → 弹窗表单

**表格列**:
| 字段 | 说明 |
|------|------|
| 岗位名称 | 关联的招聘岗位 |
| 候选人 | 面试人姓名 |
| 面试时间 | 日期时间 |
| 面试方式 | 视频/电话/线下 |
| 状态 | 待面试/已完成/已取消（badge） |
| 操作 | 进入面试/取消 |

**创建面试弹窗**:
- 选择岗位（下拉）
- 候选人姓名
- 候选人电话
- 面试时间（datetime）
- 面试方式（下拉：视频面试/电话面试/线下面试）
- 备注

### 4. 政策咨询

**搜索栏**: 按政策关键词搜索

**筛选标签**: 全部 | 劳动合同 | 社保公积金 | 劳动仲裁 | 灵活用工

**列表**:
| 字段 | 说明 |
|------|------|
| 政策标题 | 政策文件/资讯标题 |
| 分类 | 所属分类标签（badge） |
| 更新时间 | 日期 |
| 操作 | 查看详情/在线咨询 |

**在线咨询弹窗**:
- 咨询政策（自动填入）
- 咨询内容（textarea）
- 联系电话

## 技术方案

- 完全沿用 `12-hr-flagship.html` 现有架构
- 修改 `ROLE_CONFIG.org.tabs` 数组，新增一个 tab 项
- 在 `getContent` 的 `contentMap` 中添加 `'org-job'` 函数
- 复用已有 CSS 类：`.stats-grid`, `.stat-card`, `.table-wrapper`, `.filter-tabs`, `.search-bar`, `.modal-overlay` 等
- 新增 JS 函数：`publishJob()`, `createInterview()`, `consultPolicy()` 等
- 所有弹窗复用现有的 `openModal()` 模式

## 实现步骤

1. 在 `ROLE_CONFIG.org.tabs` 中插入 `org-job` tab 定义
2. 在 `contentMap` 中添加 `'org-job'` 函数生成 HTML
3. 添加岗位发布相关 JS 函数（`publishJob`, `editJob`, `viewJobDetail` 等）
4. 添加视频面试相关 JS 函数（`createInterview`, `enterInterview` 等）
5. 添加政策咨询相关 JS 函数（`viewPolicy`, `consultPolicy` 等）
6. 验证页面功能完整
