# 就业人员画像详情页 - 全屏设计规范

**日期**: 2026-05-09
**版本**: 1.0

---

## 1. 设计目标

将就业人员画像的"详情"按钮弹窗改为全屏展示，合并求职者画像和人才画像为一个统一的详情页面，按照PRD需求文档中描述的内容进行展示。

## 2. 核心变更

### 2.1 模态框改为全屏页面
- 移除原有的 `person-detail-modal` 模态框
- 创建新的全屏详情页布局
- 详情内容直接覆盖整个页面

### 2.2 合并画像类型
- 移除 Tab 切换（求职者画像/人才画像/重点群体画像）
- 所有内容在一页中按区块展示
- 根据PRD 2.8.3的描述，展示以下内容：
  - 基础信息标签 + 教育特征标签（左侧展示）
  - 就业特征标签 + 技能标签（中部展示）
  - 帮扶标签 + 意向标签（底部展示）

## 3. 布局结构

### 3.1 顶部区块（2行）

**第1行**：
- 左侧（1/3宽度）：人员基本信息卡片
  - 姓名、身份证、学历、群体类型、联系方式
- 右侧（2/3宽度）：核心评估指标（6个）
  - 教育竞争力、就业稳定性、职业技能、帮扶指数、薪资趋势、意向匹配度

**第2行**：
- 个性化服务推荐（横向排列4个服务卡片）

### 3.2 中部区块

- 左侧（1/2宽度）：就业信息卡片
  - 公司名称、职位、薪资、合同类型、工作年限、入离职记录
- 右侧（1/2宽度）：动态标签云
  - 6个分类的标签按分类聚合展示
  - 标签分类：基础信息标签、教育特征标签、就业特征标签、技能标签、帮扶标签、意向标签
  - 无需Tab切换，所有标签同时可见

### 3.3 底部区块

- AI分析洞察卡片
  - 画像分析小结
  - AI预测（就业率预测、薪资趋势预测）
  - 就业建议

## 4. PRD对应内容映射

| PRD章节 | 展示内容 |
|---------|----------|
| 2.8.3 求职者画像 | 教育竞争力、就业稳定性、帮扶指数、薪资变化趋势 |
| 2.8.3 人才画像 | 就业信息、教育经历、职业技能、就业意向 |
| 2.8.3 重点群体画像 | 群体属性、群体规模、就业率 |
| 2.8.2 动态标签体系 | 基础信息标签、教育特征标签、就业特征标签、技能标签、帮扶标签、意向标签 |
| 2.8.4 画像详情展示 | 原始数据、标签明细、画像分析 |
| 三、AI与技术应用 | 知识图谱分析、标签自动生成、AI分析洞察 |

## 5. 实现要点

### 5.1 技术实现
- 使用新的全屏容器 `.detail-fullscreen-container`
- CSS Grid 布局实现响应式分栏
- 标签按分类聚合，使用折叠面板或横向分区展示

### 5.2 数据更新
- 更新 `viewPersonDetail()` 函数，不再使用Tab切换
- 所有数据在一页中直接展示

### 5.3 权限控制
- 保持原有权限逻辑（企业用户仅查看人才相关信息）
